반응형 디자인의 브레이크 포인트
(xs) : 화면 너비 576px보다 작을 경우 해당
Sm : 화면 너비가 576px보다 클 경우
Md : 화면 너비가 768px보다 클 경우
Lg : 화면 너비가 992px보다 클 경우
Xl : 화면 너비가 1200px보다 클 경우
xs해상도가 기본 브레이크 포인트 . 기본값!
그렇기에 실제 해상도 지정 할 때
col-xs-6 이 아닌 col-6 라고 지정해주어도 됨.

반응형 레이아웃의 핵심, 브레이크포인트와 칼럼
작은 화면에서 개별 칼럼은 항상 세로로 쌓이게된다.
또한, 작은 단위에서 명령을 내린 브레이크포인트는 더 높은 단위의 브레이크 포인트에서도 동일한 방식으로 동작하게 된다.
그렇기에 화면 해상도에 따라 다른 결과를 보여주고싶다면, 별도의 브레이크 포인트를 작성해야한다.
그렇기에 col-md-6 하나만 작성하더라도 col-md-6 col-lg-6 두개를 작성한 경우와 동일한 결과를 보여준다
행(row)는 기본적으로 flex 속성을 가지고있다. 그렇기에 행 내의 다른 칼럼들은 항상 동일한 높이값을 갖게 된다.
좌우/상하 레이아웃에서는 – 자동 마진을 사용하거나 플렉스 박스(flexbox)의 align-items 나 justify content를 사용해도 문제 없다.
부트스트랩에서 쓰이는 자바스크립트에 대해 알아보자
부트스트랩의 문서를 살펴보면, UI 컴포넌트의 문서의 대부분이 스타일링된 css 컴포넌트에 대한 내용이다.
Css는 기본적으로 스타일을 꾸미는 용도로 사용되지만, 자바스크립트는 이들을 동작하게 하는 용도이다.
일부 부트스트랩의 ui 컴포넌트들은, css 기반의 규격과 스타일링만을 지원한다. 기본적으로 자바스크립트를 기반으로 한 규격이 아닌 상태이다.
예를 들어 기존에 보았던 뱃지나, 빵가루, 버튼, 카드규격, 입력창, 점보트론, 페이지네이션 등은 자바스크립트 기반의 규격을 지원하지 않는다.
이러한 규격을 제외한 다른 규격들은 자바스크립트를 통한 변형 / 포매팅 지원한다!

또한, 이러한 규격들을 지원하기 위해, 부트스트랩은 1세대 자바스크립트 라이브러리인 jquery를 필요로 한다.
부트스트랩이 자바스크립트 기반의 jquery 플러그인을 통해 만들어졌기 때문 !
이러한 자바스크립트 기반의 컴포넌트들을 다루는 방식은 크게 2 가지가 있다.

1. 데이터 속성값 통해, UI 컴포넌트 다루기
자바스크립트 컴포넌트를 다루기 위한 일반적으로 자주 사용되는 방식은, 데이터 속성값을 통한 방식이다.
이러한 방식은 추가적 자바스크립트나, jquery 코드를 필요로하지 않고 내부의 속성값만 조금 변경해주면 동작하는 형식이다.

2. Jquery 코딩을 통해 UI 컴포넌트 다루기
자바스크립트 컴포넌트를 다루기 위한 또다른 방식은 jquery를 이용하는 방식이다.
Jquery의 선택자나 요소들, 문법에 맞는 형식으로 – 개별 컴포넌트들을 커스터마이징 할수 있다.

부트스트랩의 개별 UI 컴포넌트 옵션은 항상 데이터 속성과 함께 연결되어 있다.
그렇기에 대부분의 ui 컴포넌트들은 자동으로 부트스트랩의 data-toggle= 와 같은 속성값이나, 클래스와 연결되어 있다.
그러나 일부 UI 컴포넌트들은 이와 다른 경우가 있다.
그 두가지는 바로 popover와 tooltip 이다. 이 두가지 ui 컴포넌트는 직접 jquery를 통해 인식 시켜줘야 한다.
