HOW USE UI COMPONENT

UI 컴포넌트 사용방법 부트스트랩의 UI 컴포넌트 사용법은 매우 간단하다.
그냥 부트스트랩 CSS 요소를 hTML 마크업에 맞게 작성해주면 된다. 예를 들어, primary 색상의 버튼을 하나 사용한다고 해보자.
위의 그림을 보면, Primary 버튼이 기본적인 색상과 다른 색으로 지정된걸 볼 수 있다. 그 이유는 웹 페이지가, 'thema.css' 파일을 통해 부트스트랩에 적용되었기 때문이다 ! bootstrap.min.css 이후에 적용되는 테마는 기본적인 부트스트랩 CSS 스타일에 덮어씌워진다! 여기서 테마에 대한 커스터마이징은 추후에 설명 하겠다. 위에서 봤던 버튼의 속성을 좀 더 자세히 살펴보면
버튼이 '.btn' 'btn-primary' 라는 명령에 의해 스타일링이 되고있다는 것을 알 수 있다. 그럼 이 두가지 CSS 속성을 좀 더 자세히 알아보자!
여기에서,대체 왜 버튼이 두 속성으로 나누어져 있을까?
기본적으로 부트스트랩은 기본 UI 요소를 간단하게 변형할 수 있는 형태로 설계되었기 때문이다. .btn을 .btn-danger로 변형하여 사용 가능한 것이 예시! UI 타입을 우선적으로 적용하고, 이후 해당 요소에 대한 스타일링을 별도로 진행할 수 있다는 것이다. 이런 설계방식은 부트스트랩의 CSS를 커스터마이징하기에 아주 편리한 구조로 만들어준다! 부트스트랩 UI 컴포넌트를 위해 쓴 HTML 마크업과 구조가 정확한지 어떻게 확인? 이 부분은 부트스트랩 사용시에 약간의 노력이 필요한 부분 ! 기본적으로 부트스트랩 CSS는 HTML 구조에 맞는 특정한 문법이 있음 예를 들어, 버튼(.btn / .btn-primary)를 사용할 때, HTML 문법은 그렇게 어렵지 X 그러나, NAVBAR, 카루셀 등의 구조들은 더 복잡한 형태 ! 사용하기 위해 약간의 시행착오 필요 !
https://getbootstrap.com/docs/4.3/components/alerts/
만약 더 복잡한 구조의 UI 컴포넌트를 사용해야한다면, 부트스트랩의 공식 문서를 확인하자!
